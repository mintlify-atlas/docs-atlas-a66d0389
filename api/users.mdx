## Register User

<api>POST /api/v1/users/create</api>

Create a new user account.

### Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="username" type="string" required>
  Desired username (cannot be 32 characters long)
</ParamField>

<ParamField body="password" type="string" required>
  User's password (will be hashed)
</ParamField>

### Response

<ResponseField name="id" type="uuid">
  User's unique ID
</ResponseField>

<ResponseField name="verified" type="boolean">
  Email verification status
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

### Example Request

```json
{
  "email": "user@example.com",
  "username": "johndoe",
  "password": "securePassword123"
}
```

### Error Codes

- `423`: Registration is disabled
- `409`: User already exists with that email or username
- `400`: Invalid email format

## Start Login

<api>POST /api/v1/login/start</api>

Initiate login flow and discover available authentication methods.

### Request Body

<ParamField body="email" type="string" required>
  Email address or username
</ParamField>

### Response

<ResponseField name="step_1" type="array">
  Available first-step authentication methods:
  - `PASSWORD`: Standard password
  - `PASSKEY`: WebAuthn passkey
  - `TOTP`: Time-based one-time password
</ResponseField>

<ResponseField name="step_2" type="array">
  Required second-step authentication methods (for 2FA)
</ResponseField>

<ResponseField name="session_id" type="string">
  Login session ID to use in subsequent requests
</ResponseField>

<ResponseField name="webauthn_data" type="string">
  WebAuthn challenge data (if passkey is available)
</ResponseField>

### Example Response

```json
{
  "step_1": ["PASSWORD", "PASSKEY"],
  "step_2": ["TOTP"],
  "session_id": "abc123def456",
  "webauthn_data": "{...}"
}
```

---

## Complete Login Step

<api>POST /api/v1/login/step/{step_id}</api>

Complete an authentication step (1 or 2).

### Path Parameters

<ParamField path="step_id" type="integer" required>
  Step number (1 or 2)
</ParamField>

### Query Parameters

<ParamField query="session_id" type="string" required>
  Login session ID from start_login
</ParamField>

### Request Body

<ParamField body="auth_type" type="string" required>
  Authentication type: `PASSWORD`, `PASSKEY`, `TOTP`, or `BACKUP`
</ParamField>

<ParamField body="data" type="string | object" required>
  Authentication data:
  - For PASSWORD: password string
  - For PASSKEY: WebAuthn credential object
  - For TOTP: 6-digit code
  - For BACKUP: backup code string
</ParamField>

### Response

- **Step 1 complete, Step 2 required**: Returns `202 Accepted`
- **Login complete**: Returns user session with cookies set
- **Authentication failed**: Returns `401 Unauthorized`
